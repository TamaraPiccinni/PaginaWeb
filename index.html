Cómo hacer una Página Web "Mejor" y "Más Linda"

Una guía paso a paso para dejar atrás las malas prácticas y construir sitios web modernos y profesionales.

Tema 1: La Estructura Invisible (HTML Semántico)

"Mejor" significa que la página es robusta, fácil de mantener y accesible. Esto se logra antes de pensar en los colores. Ya no usamos <div> para todo.

Mal (Antiguo): <div id="header">...</div> <div id="content">...</div> <div id="footer">...</div>
Bien (Moderno): <header>...</header> <main>...</main> <footer>...</footer>

¿Por qué? Le das un "mapa" a Google (mejora el SEO) y a los lectores de pantalla (mejora la accesibilidad).

Tema 2: Diseño Responsivo (¡Se ve bien en celulares!)

Hoy, la mayoría de usuarios navega en el celular. Si tu página se "rompe" en un celular, no está bien hecha.

Agrega siempre esta línea en tu <head>:

<meta name="viewport" content="width=device-width, initial-scale=1.0">

Luego, en tu CSS, usas "Media Queries" para cambiar estilos según el tamaño de la pantalla:

/* Estilos para PC (ej. 800px de ancho) */
.contenedor {
  width: 800px;
}

/* Cuando la pantalla sea de 600px o menos (celulares) */
@media (max-width: 600px) {
  .contenedor {
    width: 100%; /* Ocupa toda la pantalla */
  }
}

Tema 3: ¡Alinear con Flexbox! (El truco MÁGICO)

¿Cansado de que las cosas no queden donde quieres? Flexbox es la herramienta moderna. ¿Quieres centrar algo perfectamente? 

.contenedor-padre {
  display: flex;
  justify-content: center; /* Centra horizontalmente */
  align-items: center;     /* Centra verticalmente */
  min-height: 200px;       /* (Le damos algo de altura) */
}

Tema 4: Estética "Linda" y CSS Moderno

El secreto N°1 es: Menos es Más. No uses 10 colores. Define una paleta. Usamos Variables CSS (Custom Properties) para definir nuestros colores en un solo lugar. 

/* Define los colores en :root (la raíz del documento) */
:root {
  --color-texto: #333;
  --color-fondo: #FFF;
  --color-acento: #007bff;
}

/* Luego, úsalos donde quieras */
body {
  color: var(--color-texto);
  background-color: var(--color-fondo);
}
button {
  background-color: var(--color-acento);
}

¡Así es como funciona el botón de modo claro/oscuro! Simplemente cambia los valores de estas variables.

Bono Extra 1: La "Magia" de CSS

Estos efectos son fáciles y hacen que tu página se sienta profesional al instante.

1. Efecto :hover (Reacción al mouse)

El usuario debe saber que puede hacer clic. El transition hace que el cambio sea suave. 

.boton {
  background-color: var(--color-accent-blue-dark);
  /* 'transition' anima el cambio de propiedades (ej. color) */
  transition: all 0.3s ease;
}

/* ':hover' se activa cuando el mouse está encima */
.boton:hover {
  background-color: var(--color-accent-blue-light);
  transform: translateY(-2px); /* Moverlo 2px hacia arriba */
}

2. Sombras (box-shadow)

Da una sensación de "profundidad", como si los elementos flotaran.

.tarjeta {
  /* Sombra: (desplaz. X, desplaz. Y, difuminado, color) */
  box-shadow: 0px 4px 10px rgba(0, 0, 0, 0.1); 
}

Bono Extra 2: Imágenes (SVG vs JPG) y Videos

SVG (Gráfico Vectorial) vs JPG/PNG

Para logos, iconos y gráficos simples (como el de arriba), **SVG es casi siempre mejor**.

  • SVG (Vectores): Es código. No pesa nada, nunca se pixela y puedes cambiar sus colores con CSS.
  • JPG/PNG (Bitmap): Son fotos (píxeles). Pesan más, se pixelan si los agrandas y no puedes cambiar sus colores fácilmente. Úsalos solo para fotos de personas o paisajes.

Cómo poner una Imagen (JPG/PNG)

Apuntamos a la imagen que tienes en tu carpeta images/images.jpg.

Demo de Imagen (desde carpeta /images):

<!-- El 'alt' (texto alternativo) es OBLIGATORIO por accesibilidad -->
<img 
  src="images/images.jpg" 
  alt="Una imagen de código de desarrollo web."
  class="responsive-image"
>

<!-- En CSS, 'responsive-image' tendría: -->
.responsive-image {
  max-width: 100%; /* Asegura que no se salga de la pantalla */
  height: auto;    /* Mantiene la proporción */
  border-radius: 8px; /* ¡Esquinas redondeadas! */
  
  /* --- AÑADE ESTO PARA CENTRAR --- */
  display: block;
  margin: 0 auto;
}

Tema 5: Siguientes Pasos (Estructura y "Deploy")

1. Estructura de Carpetas Estándar

¡Ya lo estás haciendo! Poner tu CSS en `styles/` es el primer paso. 

/mi-proyecto-web/
  ├── index.html         (Tu página principal)
  ├── /styles/
  │   └── style.css      (Tu CSS va aquí adentro)
  ├── /images/
  │   └── images.jpg     (Tus imágenes van aquí)
  └── /scripts/
      └── main.js        (Tu JavaScript iría aquí)

Recuerda que el link en tu <head> debe ser:

<link rel="stylesheet" href="styles/style.css">

2. Publicar tu Página GRATIS (Deploy)

"Deploy" es simplemente subir tus archivos a un servidor para que cualquiera pueda verlos con un link.

La Opción Más Fácil: Netlify Drop

  1. Crea una carpeta en tu PC (ej. "mi-web").
  2. Pon tu `index.html`, y tus carpetas `styles` e `images` dentro.
  3. Ve a app.netlify.com/drop (necesitas cuenta gratis).
  4. Arrastra tu carpeta "mi-web" y suéltala en la caja punteada de la página.
  5. ¡Listo! Netlify te dará un link público.

La Opción Estándar (con GitHub): GitHub Pages

  1. Sube tu proyecto a un repositorio de GitHub.
  2. Ve a la pestaña "Settings" (Configuración) de tu repositorio.
  3. Baja a la sección "Pages" (Páginas).
  4. Elige la rama (branch) "main" y haz clic en "Save".
  5. GitHub te dará un link. Tarda 1-2 minutos en activarse.
