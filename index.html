Cómo hacer una Página Web "Mejor" y "Más Linda"

Una página "funcional" (HTML) es como los cimientos. El CSS es lo que le da la estructura (paredes) y el estilo (pintura). Aquí te explicamos cómo hacerlo bien.

Tema 1: La Estructura (HTML Semántico)

"Mejor" significa que la página es robusta, fácil de mantener y accesible para todos. Esto se logra antes de pensar en los colores.

1.1 ¡Usa HTML Semántico, en serio!

No uses <div> para todo. Dale un mapa a Google (SEO) y a los lectores de pantalla (Accesibilidad).

Mal: <div> (Título), <div> (Menú), <div> (Contenido)
Bien: <header>, <nav>, <main>, <section>, <footer>

Tema 2: Adaptación (Responsive Design)

Hoy, más gente navega en el celular que en la PC. Si tu página se "rompe" en un celular, no está bien hecha.

2.1 El Meta Viewport

Agrega esto SIEMPRE en tu <head>. Le dice al celular "no hagas zoom, usa el ancho real del dispositivo".

<meta name="viewport" content="width=device-width, initial-scale=1.0">

2.2 Media Queries

En tu CSS, puedes cambiar estilos según el tamaño de la pantalla:

/* Estilos para PC (default) */
.contenedor { width: 800px; }

/* Cuando la pantalla sea de 600px o menos (celulares) */
@media (max-width: 600px) {
  .contenedor {
    width: 100%; /* Ocupa toda la pantalla */
  }
}

Tema 3: Alineación (¡Flexbox!)

¿Cansado de que las cosas no queden donde quieres? Dejamos atrás los float. Flexbox es la herramienta moderna.

3.1 El truco MÁGICO para centrar

¿Quieres centrar algo perfectamente en medio de un contenedor? Fácil:

.contenedor-padre {
  display: flex;
  justify-content: center; /* Centra horizontalmente */
  align-items: center;    /* Centra verticalmente */
  height: 100vh; /* Ocupa toda la altura de la pantalla */
}

Tema 4: La Estética ("Más Linda")

El secreto N°1 es: Menos es Más.

4.1 Tipografía (Fuentes)

No uses "Times New Roman". Usa Google Fonts (gratis). No uses más de 2 fuentes distintas (una para títulos, otra para texto).

body {
  font-family: 'Roboto', sans-serif; /* Fuente para párrafos */
}
h1, h2 {
  font-family: 'Montserrat', sans-serif; /* Fuente para títulos */
}

4.2 Paleta de Colores

No elijas colores al azar. Define una paleta (usa Coolors). Elige un color principal, uno secundario y uno de fondo.

4.3 Espaciado (Padding y Margin)

¡Dale aire a tus elementos! No dejes que el texto toque los bordes.

  • Padding: Es el "relleno" dentro de un elemento.
  • Margin: Es el "espacio" fuera de un elemento (para separarlo de otros).

Bono Extra: La "Magia" de CSS

Estos 3 efectos son fáciles y hacen que tu página se sienta profesional al instante.

Efecto :hover (Reacción al mouse)

El usuario debe saber que puede hacer clic. :hover se activa cuando el mouse pasa por encima.

Transiciones (Suavidad)

El :hover es brusco. Con transition, hacemos que el cambio de color sea suave (ej: 0.3 segundos).

Tema 5: Siguientes Pasos (Estructura y Deploy)

¡Felicidades! Ya sabes crear una página. Ahora, veamos cómo organizarla y publicarla en Internet gratis.

5.1 Estructura de Carpetas Estándar

No es buena idea tener 50 archivos sueltos. La organización es clave. Esta es la estructura recomendada:

mi-proyecto-web/
│
├── index.html          <-- Tu página principal
│
├── styles/             <-- Carpeta para tu CSS
│   └── style.css
│
├── images/             <-- Carpeta para tus imágenes
│   └── hero.jpg
│
└── scripts/            <-- Carpeta para tu JavaScript
    └── main.js

Para que esto funcione, tendrías que actualizar el enlace en tu <head> a:

<link rel="stylesheet" href="styles/style.css">

5.2 Cómo hacer "Deploy" GRATIS (¡Sin Git!)

"Deploy" es la palabra elegante para "publicar tu sitio web". Hay servicios fantásticos que te dejan hacerlo solo arrastrando tu carpeta.

Opción A: Netlify Drop (La más fácil)

  • Prepara tu carpeta: Ten tu carpeta (ej. `mi-proyecto-web/`) lista con `index.html` y `styles/style.css` adentro.
  • Ve a la web: Abre app.netlify.com/drop.
  • Arrastra y Suelta: Arrastra tu carpeta `mi-proyecto-web` y suéltala en la página.
  • ¡Publicado! En segundos, te darán una URL (ej. `nombre-raro.netlify.app`) para compartir.

Opción B: GitHub Pages (Con la interfaz web)

  • Crea un Repositorio: En GitHub.com, crea un repositorio nuevo.
  • Sube archivos: Dentro del repo, haz clic en `Add file` > `Upload files`. Arrastra tu `index.html` y tu carpeta `styles`.
  • Activa Pages: Ve a `Settings` > `Pages`. Selecciona la rama `main` (o `master`) y haz clic en `Save`.
  • ¡Publicado! Tendrás una URL como `tu-usuario.github.io/tu-repo`. (Tarda 1-2 min).
